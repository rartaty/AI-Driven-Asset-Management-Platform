# AI-Driven Asset Management Platform

## 概要 (Overview)
本プロジェクトは、銀行APIと証券APIを統合し、AIによる意思決定と徹底したリスク管理を備えた「個人向け自動資産運用・管理プラットフォーム」です。
ユーザーはダッシュボードを通じて、現在の銀行残高、ポートフォリオの状況、AIの分析結果・レポートを1画面（1スクロール）で直感的に把握することができます。

## 運用戦略 (Trading Strategies)
本システムは、市場の変動に対して頑健性を持たせるため、目的の異なる3つの独立したエンジン（バケット）によるハイブリッド運用を行っています。

> **Note**: セキュリティおよび戦略的優位性（アルファ）保護のため、具体的な選定アルゴリズムや閾値などのトレードロジックはリポジトリから非公開（削除）としています。

1. **長期運用（サテライト枠 / テンバガー狙い）**
   - 厳格な財務スクリーニングとAIによるマクロ分析を組み合わせ、将来の圧倒的な成長が期待できる銘柄（10倍株候補）を大底圏で仕込む戦略です。
2. **長期運用（コア枠 / 堅実積立）**
   - 安定した複利効果を狙い、PBRやフリーキャッシュフロー等のファンダメンタルズを重視した安全性の高い運用を行います。
3. **短期運用（デイトレード）**
   - 機関投資家のアルゴリズムなどを検知し、短期的に利益を積み上げます。短期枠で得られた利益は、独自の「Profit Sweep（利益振替）」ロジックにより長期コア枠へと還流され、長期的な複利効果を最大化します。

## アーキテクチャ・技術要件 (Architecture & Tech Stack)
ローカルPC上でのセキュアな稼働を前提とした、モダンで軽量な構成を採用しています。

- **Frontend**: Next.js, React, TypeScript, CSS Modules (ダークモード・グラスモーフィズム対応の高品位UI)
- **Backend**: Python 3.11, FastAPI (非同期処理によるリアルタイム通信)
- **Database**: SQLite / SQLAlchemy 2.0 (コンテナ内でのファイルベース運用による高速バックアップ対応)
- **AI Engine**: ローカルLLM (Ollama経由でのGemma 2 / Codestral) を用いたオフライン推論（※使用AIは将来変更の可能性あり）
- **Infrastructure**: Podman / Docker Compose によるコンテナオーケストレーション

## インフラとセキュリティ・セーフティ設計 (Security & Safe Design)
個人の重要な金融資産を自動で扱うため、システムは何よりも「安全性（資金を守ること）」を最優先に設計しました。

- **フェイルセーフ・キルスイッチ（自動遮断）**:
  口座残高がユーザーの定めた「生活防衛費」を下回った場合や、予期せぬ大きなドローダウン（資産の急減）を検知した場合、システムが物理的に新規の「買い発注」をロックするハードウェアレベルのキルスイッチを搭載しています。
- **完全ローカルAI・ゼロトラスト環境**:
  AI推論には外部API（OpenAI等）を利用せず、ローカル環境で稼働するLLMを使用しています。これにより、自身の取引履歴やポートフォリオなどの機密情報が外部サーバーに送信されるリスクを完全に排除しています。
- **シークレット情報の分離**:
  APIキーやパスワード等の認証情報はソースコード上に一切ハードコードせず、AWS Systems Manager (SSM) Parameter Store 等のセキュアな外部ボルトから動的に取得・注入する設計を採用しています。
- **オーバーナイトリスクの排除**:
  短期トレード枠においては、市場引け（14:50）までに全ポジションを必ず強制決済し、翌日に持ち越さない（ノーポジションで終える）ことで、夜間の突発的な市況変動や海外ニュースによる暴落リスクを回避しています。
- **リアルタイム監視と通知 (Observability)**:
  DiscordのWebHook機能をシステム全体に組み込み、システムエラー、約定通知、AIの思考ログ（If-Thenレポート）などをスマートフォンへ即座に通知する堅牢な監視基盤を構築しています。
