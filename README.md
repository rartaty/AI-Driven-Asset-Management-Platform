# AI-Driven Asset Management Platform

## 概要 (Overview)
本プロジェクトは、銀行APIと証券APIを統合し、AIによる意思決定と徹底したリスク管理を備えた「個人向け自動資産運用・管理プラットフォーム」です。
ユーザーはダッシュボードを通じて、現在の銀行残高、ポートフォリオの状況、AIの分析結果・レポートを1画面（1スクロール）で直感的に把握することができます。

## 運用戦略 (Trading Strategies)
本システムは **「投資判断ファースト原則」** に基づき、AI が利益見込みありと判断した場合のみ投資を実行します（比率維持のためだけの強制配分は行いません）。証券口座を独立した複数のバケットに分け、経済・市場状況に応じて**動的にトップレベル配分を見直す**ハイブリッド運用を行っています。

> **Note**: セキュリティおよび戦略的優位性（アルファ）保護のため、具体的な選定アルゴリズム・閾値・判定ロジックは公開リポジトリから非公開（削除）としています。

1. **投資信託（Passive Core）**
   - インデックスベースのパッシブ運用枠。安定した複利効果を担う土台です。
2. **長期運用（V6 モデル）**
   - **コア枠（長期内 70%前後 / 堅実積立）**: PBR・フリーキャッシュフロー等のファンダメンタルズを重視した安全性の高い銘柄群で運用します。
   - **サテライト枠（長期内 30%前後 / テンバガー狙い）**: 厳格な財務スクリーニングと AI マクロ分析を組み合わせ、将来の圧倒的な成長が期待できる銘柄（10倍株候補）を大底圏で仕込む戦略です。
3. **短期運用（デイトレード）**
   - 機関投資家のアルゴリズムなどを検知し、短期的に利益を積み上げます。短期枠で得られた利益は、独自の「Profit Sweep（利益振替）」ロジックにより長期コア枠へと自動還流され、長期的な複利効果を最大化します。

なお、銀行口座（生活防衛費を含む現金）は AI 管理対象外の**読取専用プール**として、証券口座とは完全に独立して扱われます。

## アーキテクチャ・技術要件 (Architecture & Tech Stack)
ローカルPC上でのセキュアな稼働を前提とした、モダンで軽量な構成を採用しています。

- **Frontend**: Next.js 16 / React 19 / TypeScript / Tailwind CSS 4 (ダークモード・グラスモーフィズム対応の高品位UI)
- **Backend**: Python 3.11, FastAPI (非同期処理によるリアルタイム通信)
- **Scheduler**: APScheduler (時間ベースの自律稼働ループ)
- **Database**: SQLite / SQLAlchemy 2.0 (ファイルベース運用による高速・無停止バックアップ対応)
- **AI Engine**: ローカルLLM (Ollama 経由での Gemma 2 / Codestral) を用いたオフライン推論（※使用AIは将来変更の可能性あり）
- **Secret Management**: AWS Systems Manager (SSM) Parameter Store + KMS 暗号化
- **Infrastructure**: ローカル PC 上で直接稼働。将来的にコンテナ化（Podman / Docker Compose）を予定

## インフラとセキュリティ・セーフティ設計 (Security & Safe Design)
個人の重要な金融資産を自動で扱うため、システムは何よりも「安全性（資金を守ること）」を最優先に設計しました。

- **フェイルセーフ・キルスイッチ（自動遮断）**:
  予期せぬ大きなドローダウン（資産の急減・閾値はユーザー設定可能）を検知した場合、システムが自動的に新規買付発注をブロックするキルスイッチを搭載しています。**非対称オーバーライド設計**により、新規エントリーのみブロックし既存ポジションの決済（Exit）は許可するため、ショート建玉の買戻し決済も安全に行えます。解除は手動操作のみ（自動解除なし）。
- **完全ローカルAI・ゼロトラスト環境**:
  AI推論には外部API（OpenAI等）を利用せず、ローカル環境で稼働するLLMを使用しています。これにより、自身の取引履歴やポートフォリオなどの機密情報が外部サーバーに送信されるリスクを完全に排除しています。
- **シークレット情報の分離**:
  APIキーやパスワード等の認証情報はソースコード上に一切ハードコードせず、AWS Systems Manager (SSM) Parameter Store（KMS暗号化）から動的に取得・注入する設計を採用しています。
- **オーバーナイトリスクの排除**:
  短期トレード枠においては、市場引け（14:50）までに全ポジションを必ず強制決済し、翌日に持ち越さない（ノーポジションで終える）ことで、夜間の突発的な市況変動や海外ニュースによる暴落リスクを回避しています。
- **リアルタイム監視と通知 (Observability)**:
  DiscordのWebHook機能をシステム全体に組み込み、システムエラー、約定通知、AIの思考ログ（If-Thenレポート）などをスマートフォンへ即座に通知する監視基盤を構築しています。
